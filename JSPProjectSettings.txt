1. 개발환경

IDE Tool : Eclipse 24-09 R
JDK : JDK 21
Servlet : Servlet 5.0 이상 (사용 버전 6.0)
Apache-Tomcat : 10.1v

2. 설정

	1) JDK 설치 및 설정
	 - https://adoptium.net/temurin/releases
	 - 21.0.5+11-LTS Windows x64 버전의 JDK(.zip) 설치
	 	> C드라이브의 JDK 폴더 생성 후, 압축 해제
	 - JDK21 환경변수 설정
	 - 명령 프롬프트에서 JDK 버전 확인
	 > 'java -version' 명령어 실행
	  
	  ** JDK 21은 다운로드 경로는 오라클도 존재하고 테무린도 존재하고 다양합니다.
	  	 어떤걸 다운로드 받아서 설정하더라도 상관 없습니다.
	  	 
	 2) Apache-Tomcat 설치
	  - https://tomcat.apache.org/download-10.cgi
	  - apache-tomcat 10.1.xx 버전의 Core(.zip) 설치
	  
	 3) Eclipse 설치
	  - https://www.eclipse.org/download/packages/relaeas/2024-06/r
	   > Web Developers가 포함된 종류여야지만 웹 개발이 가능합니다.
	   
	 4) Eclipse 기본 설정
	  - Web 개발을 위한 기본 설정
	   	
	   	4-1) Encoding 설정
	   	 - Window > Preferences > 'enc' 검색
	   	 - General, Web, XML 항목들 모두 'UTF-8'로 업데이트 및 설정
	   	 
	   	4-2) Java 설정
	   	 - Windwo > Preferences > Java
	   	 - Installed JREs 항목에 JDK21 설정
	   	 
	   	4-3) Server 설정
	   	 - Window > Preferences > Server
	   	 - Runtime Enviroments 항목에 tomcat 설정
	   	 
	   	4-4) Font 설정
	   	 - Window > Preferences > 'font' 검색
	   	 - 기본 폰트는 Consolar, 하지만 띄어쓰기가 구분 될 수 있는 포느트로 설정하길 권장
	   	4-5) icon 설정
	   	 - 바로가기 만들고 icon 편집(바로가기 아이콘 마우스 우 클릭 후 속성의 아이콘 편집)
	   	 
3. JSP Project 생성
	
	1) Dynamic Web Project 생성
	- File> New, Dynamic Web Project 검색
	  > Project Name : JSPProject
	  > Target Runtime : Apache Tomcat v10.1
	  > Dynamic Web Module Version : 6.0 (Servlet 버전에 해당한다)
	  > Configuration : Apache Tomcat v10.1

	  *** Servlet 버전에 따른 특징
	  # Servlet 3.1
	  - Javax.servlet 패키지 네임스페이스 사용
	  - HttpServletRequest 가용 메서드 범위 개선 (ex, getServletContext())
	  # Servlet 5.0
	  - Javax.servlet -> jakarta.servlet으로 변경
	  - 보안 및 HTTP 향상
	  - 최소 JDK8, 권장 JDK11
	  # Servlet 6.0
	  - 최소 JDK11 이상 필요
	  - jakarta ee10  이상
	  - Deprecated API (오래된 기능 제거) ::: DiskFileUpload 객체가 제거됨
	  
	  2) Dynamic Web Project 구조 살피기
	  - src/main/java : .java 파일고 같은 Source들을 관리
	  - src/main/webapp : jsp 파일과 같은 정적 소스 관리
	  
	  		*** ClassPath와 ContextPath
	  		# ClassPath
	  		- JVM이 Java 클래스를 찾는 경로
	  		- .class파일이 위치한 폴더
	  		- 기존 프로젝트 내에서는 'src/main/java' 위치이거나 배포시에는 'WEB-INF/classes'경로
	  		# ContextPath
	  		- 웹 애플리케이션의 URL에서 루트 경로를 의미
	  		- Tomcat 배포 시 , Root 폴더가 '/'(루트)를 의미
	  		
	  3) Project 환경 설정
	  - 프로젝트 우 클릭, Properties or 프로젝트 선택 후, Alt + Enter
	  	> Java Build Path : Modulepath의 JRE System Library를 JDK21로 설정
	  	> Compiler : JDK 21 설정으로 되어 있는지 확인 (설정된 JDK 버전에 맞춰서)
	  	> Project Facets : Servlet 버전 6.0, JAva 버전 21 설정 및 Runtime 탭 클릭 후 Apache-Tomcat 체크 설정
	  	
	  4) web.xml 설정
	  - 프로젝트 우 클릭 후, Java EE Tools > Generated Deployment Stub 클릭
	   > welcom 페이지 설정 및 관련 필터 설정이 필요한 경우 web.xml 파일 살려서 설정
	   
	  5) Apache-Tomcat 설정
	  - Server 탭의 Overview 항목에서 ports를 80(기본포트)으로 설정
	  - Server 탭의 Overview 항목에서 Timeouts의 start를 120s로 설정
	  - Server 탭의 Modules 항목에서 Path를 '/'로 설정
